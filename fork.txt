In many applications, creating multiple processes is an effective method for task
decomposition. For example, a network server process can create a new sub process to
process each request while listening for client requests. At the same time, the server process
will continue to listen for more client connection request s. Decomposing tasks in such a way
usually simplifies the design of applications and improves the concurrency of the system.
(That is, more tasks or requests can be processed at the same time.)
